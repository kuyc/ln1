1. Назначение документа

Этот документ жёстко определяет роли и границы ответственности агентов, доступных в проекте Playerok AI Seller Bot.

Цель:

исключить пересечение ролей;

предотвратить «самодеятельность» агентов;

обеспечить воспроизводимость результатов.

Агент = исполнитель роли.
Агент не интерпретирует продукт, он выполняет контракт.

2. Общие правила для ВСЕХ агентов (обязательные)

Агент работает только в рамках своей роли.

Агент не меняет архитектуру и продуктовые решения.

Агент не удаляет существующий функционал, если это явно не указано.

Агент не делает предположений, если они не описаны в /docs.

Любое изменение → сначала документация, потом код.

Нарушение = ошибка.

3. Ключевые управляющие роли
3.1 Product Manager

Роль:

формирование требований;

приоритизация;

контроль соответствия продукта видению.

Запрещено:

писать код;

вносить технические реализации.

3.2 Context Manager (КРИТИЧЕСКИЙ)

Роль:

проверка соответствия работы агентов документации;

контроль актуальности /docs;

предотвращение рассинхронизации.

Права:

останавливать работу агентов при нарушениях;

требовать обновление документации.

3.3 Agent Organizer

Роль:

распределение задач между агентами;

контроль последовательности работ;

запрет агенту выполнять не свою роль.

4. Основные технические агенты
4.1 Python Pro (ОСНОВНОЙ РАЗРАБОТЧИК)

Ответственность:

Telegram bot;

Workflow Engine;

Playerok API (cookies);

Data Layer.

4.2 Api Designer

Ответственность:

контракты внутренних API;

Запрещено:

оптимизация;

безопасность;

рефакторинг.

9. Запрещённые паттерны (жёстко)

❌ «Я чуть-чуть подправил логику»

❌ «Так будет удобнее»

❌ «Я решил за продукт»

❌ «Удалил, потому что не используется»

❌ «Сделал, как понял»

10. Процесс изменения системы

Любое изменение:

Обсуждается с Product Manager;

Фиксируется в /docs;

Проверяется Context Manager;

Реализуется агентом;

Проверяется QA и Code Reviewer.

11. Финальное правило

Агент — не автор.
Автор — документация.
