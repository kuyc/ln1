1. Назначение документа

Этот документ формально определяет режимы работы проекта ln1
и их влияние на поведение Workflow Engine.

Режимы:

определяют допустимые переходы состояний;

определяют уровень автономности системы;

не могут трактоваться свободно.

Если поведение системы не соответствует этому документу —
➡️ это ошибка реализации.

2. Общие правила режимов (НЕ НАРУШАЮТСЯ)

Режим выбирается только администратором

Режим фиксируется при старте Workflow

Режим нельзя изменить «на лету»

Любой режим можно остановить вручную

При ошибке режим прекращается

3. Список поддерживаемых режимов

Проект ln1 поддерживает ровно два режима:

Semi-Auto (полуавтономный)

Auto (автономный)

Любые другие режимы:

не поддерживаются;

считаются вне рамок проекта.

4. Semi-Auto режим (ПОЛУАВТОНОМНЫЙ)
4.1 Назначение

Semi-Auto режим предназначен для:

безопасного использования;

тестирования;

контролируемых продаж;

ручного масштабирования.

Это базовый режим проекта.

4.2 Характеристики Semi-Auto

запуск: вручную администратором;

генерация товара: один раз;

обязательное подтверждение администратора;

после продажи Workflow завершается;

автоматическое перевыставление запрещено.

4.3 Поведение Workflow
INIT
 → DATA_COLLECTION
 → CONTENT_GENERATION
 → PRICE_CALCULATION
 → ADMIN_CONFIRMATION
 → PUBLISHING
 → WAIT_FOR_SALE
 → POST_SALE
 → STOPPED

4.4 Ограничения Semi-Auto

Запрещено:

автоматическое перевыставление;

повторный цикл без нового запуска;

обход подтверждения администратора.

5. Auto режим (АВТОНОМНЫЙ)
5.1 Назначение

Auto режим предназначен для:

масштабирования продаж;

повторяемых товаров;

снижения ручного участия.

Auto режим всегда считается повышенным риском.

5.2 Условия включения Auto режима

Auto режим может быть включён ТОЛЬКО если:

Semi-Auto уже использовался;

нет критических ошибок;

администратор осознанно подтвердил запуск.

5.3 Характеристики Auto режима

запуск: вручную администратором;

первый цикл требует подтверждения;

последующие циклы — без подтверждения;

после каждой продажи товар перевыставляется;

цена может корректироваться автоматически;

режим работает до ручной остановки.

5.4 Поведение Workflow
INIT
 → DATA_COLLECTION
 → CONTENT_GENERATION
 → PRICE_CALCULATION
 → ADMIN_CONFIRMATION (только 1 раз)
 → PUBLISHING
 → WAIT_FOR_SALE
 → POST_SALE
 → RELIST
 → DATA_COLLECTION
 → ...

5.5 Ограничения Auto режима

Запрещено:

запуск без явного подтверждения;

самовосстановление после ошибки;

бесконечная активность без пауз;

обход лимитов цены и частоты.

Любая критическая ошибка → ERROR → остановка Auto режима.

6. Сравнение режимов
Параметр	Semi-Auto	Auto
Запуск	Вручную	Вручную
Подтверждение	Всегда	Только 1 раз
Перевыставление	❌	✅
Цикличность	❌	✅
Риск	Низкий	Повышенный
Рекомендуется	Да	После опыта
7. Ручная остановка режима

В любом режиме администратор может:

остановить Workflow;

отключить Auto режим;

перевести систему в STOPPED.

Остановка:

имеет приоритет над всеми действиями;

не может быть проигнорирована.

8. Ошибки и режимы

При любой ошибке:

Workflow переходит в ERROR;

текущий режим прекращается;

требуется ручное вмешательство.

9. Финальное правило режимов

Semi-Auto — режим по умолчанию.
Auto — инструмент масштабирования, а не стартовая точка.
