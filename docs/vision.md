1. Назначение документа

Этот документ определяет видение проекта ln1 и служит
стратегической и концептуальной основой для всей разработки.

Документ обязателен для:

всех агентов;

всех разработчиков;

любых решений по архитектуре, логике и автоматизации.

Если решение противоречит этому документу —
➡️ решение считается ошибочным.

2. Что такое ln1

ln1 — это Telegram-бот для автоматизации продажи цифровых товаров
на платформе Playerok с использованием:

неофициального Playerok API (через cookies);

нейросетей для генерации контента и аналитики;

управляемых workflow с жёстким контролем.

Проект не является официальным продуктом Playerok.

3. Ключевая цель проекта
Главная цель

Максимально автоматизировать процесс продажи товаров на Playerok
без потери контроля, безопасности и управляемости.

Автоматизация не является самоцелью.
Контроль администратора — приоритет.

4. Основная ценность проекта
Для администратора

экономия времени;

снижение ручной работы;

системное масштабирование продаж;

контроль над каждым этапом.

Для системы

повторяемые процессы;

воспроизводимость;

защита от ошибок и банов;

предсказуемое поведение.

5. Базовые принципы (НЕ НАРУШАЮТСЯ)

Контроль важнее автономности

Безопасность важнее прибыли

Документация важнее кода

Источник истины — архив проекта

Лучше остановиться, чем продолжить с риском

6. Роль нейросетей в проекте

Нейросети в ln1 — исполнители, а не управляющие.

Нейросети используются для:

генерации названий, описаний и комментариев товаров;

генерации превью изображений;

анализа рынка и цен;

анализа спроса и продаж.

Нейросети НЕ МОГУТ:

самостоятельно включать или отключать режимы;

обходить ограничения;

принимать финальные бизнес-решения;

действовать без подтверждения, если оно требуется.

7. Поддерживаемые режимы работы (концептуально)

Проект поддерживает два принципиальных режима:

7.1 Полуавтономный режим

действия запускаются администратором;

генерация товара происходит один раз;

публикация требует подтверждения;

после продажи workflow завершается.

7.2 Автономный режим

запускается администратором;

первый товар требует подтверждения;

после каждой продажи товар перевыставляется автоматически;

автономность может быть остановлена в любой момент.

Подробная логика режимов описана в modes.md.

8. Ценообразование (концептуально)

Цена товара:

берётся из базы данных или

рассчитывается нейросетью на основе анализа Playerok.

Система:

отслеживает спрос;

корректирует цену в допустимых пределах;

никогда не выходит за заданные лимиты.

Детальная логика описана в pricing.md.

9. Управление рисками

Проект изначально проектируется как risk-aware система:

лимиты действий;

rate limit;

паузы;

ручные подтверждения;

аварийная остановка.

Автоматизация всегда обратима.

10. Границы проекта

Проект НЕ:

гарантирует доход;

обходит законы;

выдаёт себя за официальный сервис Playerok;

действует скрытно от администратора.

11. Эволюция проекта

Проект ln1:

развивается итеративно;

допускает расширение логики;

допускает улучшение нейросетей;

не допускает слом существующих принципов.

Любое развитие:

сначала фиксируется в документации;

затем реализуется в коде;

затем тестируется.

12. Финальное утверждение

ln1 — это управляемая автоматизация.
Автономность — инструмент, а не цель.
Контроль — основа системы.
